# Changelog

## 1.6.3

### 新增

- 新增 DuckDB 启动阶段 WAL 回放异常恢复能力，在检测到 WAL replay 失败时尝试清理 WAL 并重新打开数据库
- 新增 Windows 关机 / 注销会话结束处理，在系统结束会话时触发短退出流程，尽量完成数据保存与数据库关闭

### 优化

- 优化数据库退出流程，关闭前执行强制 checkpoint，并记录退出阶段耗时与错误日志，降低异常退出后的数据库恢复风险
- 优化游戏库与分类详情页的数量展示，筛选状态下显示当前筛选结果数量与状态名称
- 优化统计页与游戏详情统计图在无游玩数据时的坐标轴展示，避免空图表比例异常
- 优化统计总时长展示为紧凑格式，减少窄空间下的文字溢出

### 修复

- 修复下拉选择菜单在部分弹窗或浮层中被遮挡的问题

## 1.6.2

### 新增

- 新增 Bangumi 账号授权与元数据接入，支持在设置页直接连接账号，并将 Bangumi 作为元数据来源参与搜索、导入匹配与刷新
- 新增本地 MCP 服务，支持通过本地端点向外部 AI / 工具开放游戏列表、详情、游玩记录、元数据搜索、统计与启动能力
- 新增 Windows 开机自启动选项，支持随系统登录后台启动 LunaBox
- 新增应用数据面板，支持一键打开数据目录与导出日志 ZIP

### 优化

- 重构弹窗渲染层，统一通过 Portal 挂载多个模态框，改善层级、遮罩与滚动体验
- 优化游戏库、分类列表与分类详情页的搜索和排序记忆，减少重复设置筛选条件
- 优化统计页图表标签与时长格式，可读性更好

### 修复

- 修复 SlideButton 在容器尺寸变化时滑块位置不同步的问题，提升切换控件稳定性（#59 ）
